# ttt-course-builder

将已有的学习文档转化为符合 **TTT（Train the Trainer）**标准的完整培训课程材料。

## 功能概览

一条指令，输入学习文档，自动输出四份课程材料：

| 输出文件 | 内容 |
|---------|------|
| `课程大纲.md` | 学习目标、逻辑主线、时间表、配套材料清单 |
| `讲师手册.md` | 完整教案 + 逐字话术 + 授课技巧标注 + 正反误区对比 |
| `学员手册.md` | 一页纸速查摘要 + 空白练习模板（适合打印） |
| `练习题.md` | 场景化练习 + 评分标准 + 时间/形式说明 |

## 触发方式

在 Claude Code 中，以下任意表述均可触发本 skill：

- "开发培训课程"
- "做课程开发"
- "把学习文档变成培训材料"
- "TTT 课程开发"
- "出培训教案"
- "做讲师手册"

## 工作流程

```
第一步：信息收集（3 个问题）
    ↓ 目标学员 / 课程范围 / 讲师角色
第二步：分析学习文档
    ↓ 提炼框架 + 评估素材 + 设计逻辑主线
第三步：生成四个输出文件
    ↓ 大纲 + 讲师手册 + 学员手册 + 练习题
第四步：TTT 合规性自检
    ↓ 对照 TTT 第 2、3 步标准清单验收
```

## TTT 标准说明

**TTT 第 2 步（课程开发）**：  
拆解任务 → 提炼要点 → 设计案例（真实场景 + 正反对比）→ 设计练习

**TTT 第 3 步（授课技巧）**：  
开场话术、金字塔原理讲解结构、互动节点、案例设计、收尾行动计划——均与教案内容一一对应标注

## 讲师手册质量标准

讲师手册是本 skill 的核心交付物，每个模块必须包含：

- `【授课技巧】` 标注（案例驱动法 / 对比纠错法 / 现场演示法 / 故事冲突法等）
- 所有表格、案例数据直接写入，不用"展示 XX"占位
- 误区以 ❌/✅ 正反对比格式呈现
- 关键话术用引用块 `>` 标注
- 互动节点明确说明时长和操作方式

## 安装

### 方式一：ClawdHub（推荐）

```bash
# 如已配置 Claude Code CLI
claude skill install ttt-course-builder
```

### 方式二：手动安装

1. 下载本仓库
2. 将 `ttt-course-builder/` 文件夹复制到 `~/.claude/skills/`
3. 重启 Claude Code

## 使用示例

```
用户：帮我把这份《销售漏斗管理方法论》文档做成培训课程材料

Claude：好的，开始前先确认三个问题：
1. 目标学员是谁？（管理层 / 骨干员工 / 混合班）
2. 课程覆盖范围？（系列课 / 单门课 3-4h / 专题课 1-2h）
3. 你的讲师角色？（主讲培训师 / 课程设计者）
```

回答后，自动在指定目录生成 `课程开发/` 文件夹及四份课程材料。

## 适用场景

- 企业内训课程开发
- 知识沉淀 → 可复制的培训体系
- 带新人 / 传帮带场景的教案设计
- 外部培训师交付标准化课程包

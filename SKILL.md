# TTT 课程开发 Skill

将已有的学习文档转化为符合 TTT（Train the Trainer）标准的完整培训课程材料。

## 什么是 TTT 标准

TTT 第 2 步（课程开发）要求：
- 拆解任务 → 提炼要点 → 设计案例（真实场景 + 正反对比）→ 设计练习
- 输出物：课程大纲 + 讲师手册 + 学员手册 + 练习题

TTT 第 3 步（授课技巧）要求：
- 开场话术、讲解结构（金字塔原理）、互动节点、案例设计、收尾行动计划
- 授课技巧必须与教案内容一一对应标注

---

## 第一步：信息收集（开始前必做）

在开始课程开发之前，先向用户确认以下三个问题：

```
1. 目标学员是谁？
   - 管理层（关注战略决策、汇报）
   - 骨干员工（关注实操工具、流程）
   - 混合班（两者都有）

2. 课程覆盖范围？
   - 全部文档内容（系列课）
   - 精选核心内容（单门课，3-4 小时）
   - 围绕特定主题（专题课，1-2 小时）

3. 讲师角色？
   - 主讲培训师（自己授课，需要完整话术）
   - 课程设计者（交给他人授课，需要详细说明）
```

根据回答调整课程设计策略：
- 混合班 → 每个知识点设计双层解读（管理层视角 + 骨干视角）
- 单门课 → 精选 4-6 个核心框架，确保逻辑主线清晰
- 主讲培训师 → 讲师手册包含逐字话术

---

## 第二步：分析学习文档

读取用户提供的学习文档，提取以下信息：

**内容清单**：
- 有哪些框架/方法/知识点？
- 每个知识点的结构是什么（定义、步骤、案例、误区）？
- 知识点之间的逻辑关系是什么？

**可用素材评估**：
- 是否有真实案例？（可直接用于课程）
- 是否有误区说明？（可转化为正反对比）
- 是否有操作步骤？（可转化为练习题）

**课程逻辑主线设计**：
- 找到知识点之间的递进关系
- 设计一条"从问题到解决"的叙事主线
- 确保每个模块都回答一个核心问题

## 第三步：生成四个输出文件

在用户指定的目录下创建 `课程开发/` 文件夹，生成以下四个文件：

### 文件 1：课程大纲.md

包含：
- 课程名称（格式：《XX工作坊：从XX到XX》）
- 学习目标（3-4 条，用"能够..."句式，管理层和骨干都适用）
- 课程逻辑主线（用箭头图展示知识点串联关系）
- 时间表（含模块名称、框架、时长）
- 配套材料清单

### 文件 2：讲师手册.md（核心文件）

这是最重要的文件，必须包含**完整内容**，不能用"展示XX表格"代替实际内容。

每个模块的结构：

```
## 模块 N：[模块名]（XX 分钟）

**核心问题**：[这个模块回答什么问题？]

【授课技巧】[用什么讲法？为什么？]
- 案例驱动法：先展示完整结果，再倒推方法
- 对比纠错法：先展示错误做法，再给正确做法
- 现场演示法：讲师现场操作，边做边讲思考过程
- 故事冲突法：用真实冲突场景引入，制造认知张力

### ① [知识点名称]（X 分钟）

【授课技巧】[此处具体用什么技巧，一句话说明]

**[核心概念]**：[一句话定义]

[完整的表格/图示/框架内容]（标注"PPT 素材"）

**[话术/过渡语]**：
> "[具体话术文本]"

**互动节点**（X 分钟）：
> "[具体提问话术]"
[互动方式说明]

### ② [下一个知识点]...

### [案例名称]（X 分钟）

[完整案例内容，包含所有数据和结论]

**结论话术**：
> "[讲师讲解结论时的话术]"

### 常见误区

❌ **误区 1：[误区名称]**
错误做法：[具体错误行为]
✅ 正确做法：[具体正确行为]
为什么：[原因说明]
```

**讲师手册的质量标准**：
- 每个知识点都有 `【授课技巧】` 标注
- 所有表格、案例数据直接写入，不用"展示XX"代替
- 所有误区用 ❌/✅ 正反对比格式
- 关键话术用引用块（`>`）标注
- 互动节点明确说明时长和操作方式

### 文件 3：学员手册.md

包含：
- 每个框架的一页纸速查摘要（定义、核心工具、操作要点、最常见误区）
- 空白练习模板（学员课中填写用）
- 格式简洁，适合打印

### 文件 4：练习题.md

每道练习题包含：
- 对应模块
- 时间和形式（个人/小组/全场）
- 背景场景（真实、具体）
- 任务说明
- 交付物格式（模板）
- 评分标准（好的答案 vs 需要改进的答案）

练习题设计原则：
- 场景来自学习文档中的真实案例，不虚构
- 每道题对应一个核心技能点
- 难度递进：从单一框架应用到综合分析

## 第四步：TTT 合规性自检

生成完所有文件后，对照以下清单自检：

**TTT 第 2 步（课程开发）**：
- [ ] 课程大纲有明确的学习目标（用"能够..."句式）
- [ ] 讲师手册包含完整内容（不是提示稿）
- [ ] 每个案例有正反对比（❌/✅ 格式）
- [ ] 学员手册有空白练习模板
- [ ] 练习题有评分标准

**TTT 第 3 步（授课技巧）**：
- [ ] 开场有逐字话术（制造认知冲突）
- [ ] 每个知识点有 `【授课技巧】` 标注
- [ ] 互动节点明确（时间、方式、提问话术）
- [ ] 收尾有行动承诺环节
- [ ] 授课技巧与教案内容一一对应

如有缺失，补充完整后再交付。

## 参考：授课技巧选择指南

| 内容类型 | 推荐技巧 | 原因 |
|---------|---------|------|
| 框架定义 | 先结论后展开（金字塔原理） | 让学员先知道"这是什么"，再理解细节 |
| 操作步骤 | 现场演示法 | 展示思维过程，不只是结果 |
| 案例分析 | 案例驱动法 | 先看完整结果，再倒推方法 |
| 误区纠正 | 对比纠错法 | 先展示错误，产生共鸣，再给正确做法 |
| 指标/数据 | 故事冲突法 | 用数据异常制造冲突，引发思考 |
| 框架串联 | 拼图法 | 把各模块比作碎片，最后拼成完整图 |

---

## 参考：互动节点设计

| 互动类型 | 适用时机 | 时长 |
|---------|---------|------|
| 举手投票 | 引入新概念前，了解学员认知 | 1-2 分钟 |
| 即时练习 | 讲完一个工具后，立即应用 | 3-5 分钟 |
| 小组讨论 | 案例分析后，深化理解 | 5-8 分钟 |
| 小组展示 | 练习完成后，分享成果 | 7-10 分钟 |
| 全场讨论 | 有争议性话题时，激发共鸣 | 2-3 分钟 |
| 行动承诺 | 课程收尾，写便签贴墙 | 5 分钟 |
